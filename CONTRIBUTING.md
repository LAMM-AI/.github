# Como contribuir

Obrigado por considerar contribuir com a **LEMM.IA**.

Este arquivo é um *default* da organização: ele pode ser herdado por repositórios que não tiverem um `CONTRIBUTING.md` próprio.

## Código de conduta
Ao participar, você concorda em seguir o [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md).

## Antes de abrir Issue
- Verifique se já existe uma issue parecida.
- Prefira propor mudanças via issue antes de enviar PR grande.
- **Segurança:** não reporte vulnerabilidades em público — veja [`SECURITY.md`](SECURITY.md).

## Fluxo padrão (Issue → PR)
1. Abra uma Issue (bug/feature) descrevendo o problema e o contexto.
2. Faça um fork (ou branch no repositório, se tiver permissão).
3. Crie uma branch com um nome descritivo, por exemplo: `feat/nome-curto`, `fix/nome-curto`, `docs/nome-curto`.
4. Faça commits pequenos e revisáveis.
5. Abra um Pull Request usando o template (checklist + contexto).

## Padrão de commits (recomendado)
Sugerimos **Conventional Commits**:
- `feat: ...` nova funcionalidade
- `fix: ...` correção
- `docs: ...` documentação
- `refactor: ...` refatoração sem mudança de comportamento
- `test: ...` testes
- `chore: ...` tarefas internas (CI, deps, etc.)

Exemplos:
- `feat: adiciona baseline reprodutível do modelo X`
- `fix: corrige seed para reprodutibilidade`
- `docs: documenta métrica e protocolo de avaliação`

## Qualidade mínima
- Não quebre builds/testes existentes (quando houver CI).
- Inclua evidências e/ou testes para mudanças relevantes.
- Atualize documentação quando mudar comportamento, métricas ou interfaces.

## Licenças e autoria
- Garanta que você tem direito de contribuir com o conteúdo (código, texto, imagens, dados).
- Se usar material de terceiros, cite a fonte e respeite a licença.
